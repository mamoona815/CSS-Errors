CSS Errors

Common Mistakes in CSS and How to Avoid Them

1. Missing Semicolons

Forgetting a semicolon at the end of a property can break your style rule.

❌ Wrong

.bad {
  color: red
  background-color: blue;
}

✅ Correct

.good {
  color: red;
  background-color: blue;
}

2. Invalid Property Names

Misspelled or non-existent properties are ignored by the browser.

❌ Wrong

.bad {
  colr: blue;
  font-size: 16px;
}

✅ Correct

.good {
  color: blue;
  font-size: 16px;
}

3. Invalid Values

A valid property but invalid value will also be ignored.

❌ Wrong

.bad {
  width: -100px;
  color: green;
}

✅ Correct

.good {
  width: 100px;
  color: green;
}

4. Unclosed Braces

If a curly brace } is missing, the rule becomes invalid.

❌ Wrong

.bad {
  padding: 20px;
  margin: 10px;

✅ Correct

.good {
  padding: 20px;
  margin: 10px;
}

5. Extra Colons or Braces

Extra symbols such as double colons cause CSS to break.

❌ Wrong

.bad {
  color:: blue;
}

✅ Correct

.good {
  color: blue;
}
